## How to Contribute

### Reporting Bugs

If you find a bug or error:

1. Check if the issue already exists
2. Open a new issue with:
   - Clear description of the bug
   - Steps to reproduce
   - Expected vs. actual behavior
   - Your environment (OS, Python version, etc.)

### Suggesting Improvements

We welcome suggestions for:
- Better explanations
- Additional exercises
- Code improvements
- Documentation updates
- New resources

Open an issue with the `enhancement` label.

### Contributing Code

1. **Fork the repository**
2. **Create a branch** for your changes:
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. **Make your changes**
   - Follow code style (see below)
   - Add comments where helpful
   - Test your changes
4. **Commit your changes**:
   ```bash
   git commit -m "Add: Description of changes"
   ```
5. **Push and create a Pull Request**

## Code Style Guidelines

### Python Code
- Follow PEP 8 style guide
- Use meaningful variable names
- Add docstrings to functions
- Keep functions focused and small
- Comment complex logic

### Jupyter Notebooks
- Use clear markdown cells to explain concepts
- Keep code cells focused
- Include expected outputs
- Clean up output before committing

### Documentation
- Use clear, concise language
- Add examples where helpful
- Keep formatting consistent

## Pull Request Process

1. Update README.md if needed
2. Add tests if applicable
3. Ensure all notebooks run without errors
4. Update documentation
5. Submit PR with clear description

## What to Contribute

### High Priority
- Fixing bugs and errors
- Improving explanations
- Adding missing exercises
- Correcting code examples

### Medium Priority
- Additional resources
- Code optimizations
- Visualization improvements
- Translation (if you speak another language)

### Nice to Have
- New project ideas
- Additional datasets
- Performance improvements
- Style enhancements

## Code of Conduct

- Be respectful and inclusive
- Provide constructive feedback
- Help others learn
- Follow community guidelines

## Questions?

If you're unsure about anything:
- Open an issue with the `question` label
- Check existing issues/PRs
- Ask in discussions
